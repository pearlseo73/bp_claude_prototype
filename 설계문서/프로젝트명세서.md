# 프로젝트 명세서

## 1.1 목표
본 프로젝트는 **자린고치**라는 이름의 절약형 가계부 웹 서비스를 구현하는 것을 목표로 한다.  
자린고치는 일반적인 가계부 기능에 더해, **주간 예산 중심의 소비 관리**, **예정 지출 반영**, **주/월간 레포트**, **굴비 캐릭터 육성**, **메이트 비교**, **커뮤니티 기능**을 제공하는 게이미피케이션 기반 서비스이다.

사용자는 수입/지출을 기록하고, 예산 대비 현재 소비 상태를 직관적으로 확인하며, 절약 행동을 지속함으로써 자신의 굴비 캐릭터를 성장시키고 보상을 획득할 수 있다.

## 1.2 준비사항

### 1) 프로젝트 구조
Servlet + JSP 기반 Web 애플리케이션으로 구현하는 것을 목표로 한다.

#### Controller
- 클라이언트의 HTTP 요청을 처리한다.
- 요청 URL에 따라 적절한 Service를 호출한다.
- 처리 결과를 JSP(View)로 전달한다.

#### Service
- 도메인 규칙과 비즈니스 로직을 처리한다.
- Repository를 호출하여 데이터를 조회/처리한다.
- 입력 데이터에 대한 비즈니스 검증을 수행한다.
- 예산 계산, 예정 지출 계산, 굴비 성장/보상 판정, 레포트 생성 조건 처리 등을 담당한다.

#### Repository(DAO)
- 데이터 저장/조회/수정/삭제를 담당한다.

#### DTO / Domain
- 사용자, 거래 내역, 예산, 예정 지출, 굴비, 의상, 새끼 굴비, 메이트, 게시글, 댓글, 대화 로그 등의 도메인 엔티티를 표현한다.

#### View (JSP)
- 사용자 화면을 구성한다.
- EL과 JSTL을 활용하여 서버에서 전달된 데이터를 출력한다.
- 입력 폼, 목록/상세 출력, 오류 메시지, 대시보드, 통계, 레포트, 커뮤니티 화면 등을 담당한다.
- 공통 레이아웃(헤더/푸터/네비) 구성을 위해 include를 활용할 수 있다.

#### (선택) Filter
- 로그인 인증, 권한 체크, 인코딩 처리, 공통 전처리/후처리를 수행한다.

## 1.3 작업 순서
1. 팀원과 함께 기본 기능, 추가 기능, 심화 기능 요구사항을 검토하고 역할을 분배한다.  
2. 도메인에 맞는 핵심 데이터 항목을 정의한다.  
3. 기능 목록을 기반으로 화면 목록(JSP)과 URL 목록을 정의한다.  
4. MVC 구조에 맞는 패키지 구조를 설계한다.  
5. 요청 흐름을 정리하고 Front Controller 구조를 적용한다.  
6. 생성형 AI를 활용하여 클래스 다이어그램 및 URL 설계의 개선 포인트를 정리한다.  
7. 기본 기능을 구현한다.  
8. 추가 기능을 구현한다.  
9. 심화 기능을 구현한다.  
10. 프로젝트를 테스트하고, 결과물을 제출한다.

## 1.4 요구사항
본 프로젝트는 맞춤형 절약형 가계부 서비스(이하 “자린고치”라 한다.)를 구현한다.  
자린고치는 수입/지출 내역 관리 기능, 거래 내역 조회 기능, 주간/월간 예산 설정 기능, 예정 지출 관리 기능, 굴비 캐릭터 알림 기능, 주/월간 레포트 기능, 굴비 성장 및 보상 기능, 메이트 경쟁 기능, 커뮤니티 기능 등을 제공하는 웹 서비스를 구현하는 것을 목표로 한다.
